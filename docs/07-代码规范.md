# AI Cost Guard - 代码规范

## 一、分层架构规范

```
Controller  →  接收请求、参数校验、调用 Service、返回 Result<T>
Service     →  业务逻辑、事务控制、调用 Mapper
Mapper      →  数据访问，继承 BaseMapper<T>
Entity      →  数据库映射，@TableName + @TableId + @TableField
DTO         →  请求/响应数据传输对象，@Valid 校验
```

### 各层职责边界

| 层 | 允许做 | 不允许做 |
|---|---|---|
| Controller | 参数校验、调用 Service、封装 Result | 写业务逻辑、直接操作数据库 |
| Service | 业务逻辑、事务、调用多个 Mapper | 操作 HttpServletRequest、返回 Result |
| Mapper | SQL 查询（MyBatis-Plus 或 @Select） | 包含业务逻辑 |
| DTO | 参数校验注解 | 包含业务方法 |
| Entity | 表映射、字段注解 | 包含业务方法 |

## 二、包命名规范

```
com.aicostguard
├── common.*              # 通用代码，不依赖任何 module
├── infrastructure.*      # 基础设施（AOP、拦截器），可依赖 common
└── module.{模块名}.*     # 业务模块，可依赖 common/infrastructure
    ├── controller
    ├── dto
    ├── entity
    ├── mapper
    ├── service
    │   └── impl
    └── task              # 定时任务（如有）
```

**模块间依赖规则**：
- `common` 不依赖任何 module
- module 之间可以通过 Service 接口互相调用（不直接调 Mapper）
- `proxy` 模块可以调用其他模块的 Service（它是核心网关）

## 三、命名规范

### Java 命名

| 类型 | 规范 | 示例 |
|---|---|---|
| 类名 | 大驼峰 | `SysUserServiceImpl` |
| 方法名 | 小驼峰 | `createUser()` |
| 变量名 | 小驼峰 | `proxyKey` |
| 常量 | 全大写下划线 | `REDIS_TOKEN_KEY` |
| 包名 | 全小写 | `com.aicostguard.module.system` |
| Entity | 与表名对应，大驼峰 | `LlmKeyPool` → `llm_key_pool` |
| DTO 请求 | `XxxRequest` | `LoginRequest` |
| DTO 响应 | `XxxResponse` / `XxxVO` | `LoginResponse`, `DailyStatsVO` |
| Controller | `XxxController` | `AuthController` |
| Service 接口 | `XxxService` | `AuthService` |
| Service 实现 | `XxxServiceImpl` | `AuthServiceImpl` |
| Mapper | `XxxMapper` | `SysUserMapper` |

### 数据库命名

| 类型 | 规范 | 示例 |
|---|---|---|
| 表名 | 小写下划线 | `sys_user`, `llm_key_pool` |
| 字段名 | 小写下划线 | `created_at`, `proxy_key` |
| 索引名 | `idx_字段` / `uk_字段` | `idx_team_id`, `uk_user_role` |
| 逻辑删除 | 统一 `deleted` 字段 | 0=正常, 1=删除 |
| 时间字段 | `created_at` / `updated_at` | 自动填充 |

### API 路径

| 规范 | 示例 |
|---|---|
| RESTful 风格 | `GET /api/system/users` |
| 小写中划线 | `/api/apikey/pool` |
| 模块前缀 | `/api/{模块}/` |
| 代理接口 | `/api/v1/chat/completions` |

## 四、统一响应规范

所有管理接口返回 `Result<T>`：

```java
// 成功
Result.success()          // 无数据
Result.success(data)      // 有数据

// 失败
Result.fail(ResultCode.NOT_FOUND)
Result.fail(ResultCode.BAD_REQUEST, "自定义消息")

// 业务异常抛出
throw new BusinessException(ResultCode.USER_NOT_FOUND);
throw new BusinessException(ResultCode.NOT_FOUND, "项目不存在");
```

**不要**在 Controller 里 try-catch 然后手动返回错误，交给 `GlobalExceptionHandler` 统一处理。

## 五、异常处理规范

```
BusinessException（业务异常）
  └── GlobalExceptionHandler 捕获 → Result.fail(code, msg)

MethodArgumentNotValidException（参数校验失败）
  └── GlobalExceptionHandler 捕获 → Result.fail(400, 第一条错误消息)

Exception（未知异常）
  └── GlobalExceptionHandler 捕获 → Result.fail(500, "服务器内部错误")
```

**规范**：
- 可预期的业务错误 → 抛 `BusinessException`
- 不要 catch 后吞掉异常（除非有明确理由）
- 不要在 Service 里返回 null 表示失败，应该抛异常

## 六、数据校验规范

在 DTO 上使用 Jakarta Validation 注解：

```java
@NotBlank(message = "用户名不能为空")
@Size(min = 3, max = 50, message = "用户名长度3-50位")
private String username;
```

Controller 方法参数加 `@Valid`：
```java
public Result<Void> create(@Valid @RequestBody UserCreateRequest request)
```

## 七、MyBatis-Plus 使用规范

### 通用约定
- Mapper 继承 `BaseMapper<T>`，优先用内置 CRUD 方法
- 复杂查询用 `LambdaQueryWrapper`，不要用字符串字段名
- 自定义 SQL 用 `@Select` 注解或 mapper XML

```java
// 推荐：Lambda 方式
LambdaQueryWrapper<SysUser> wrapper = new LambdaQueryWrapper<>();
wrapper.eq(SysUser::getUsername, username);

// 不推荐：字符串方式
QueryWrapper<SysUser> wrapper = new QueryWrapper<>();
wrapper.eq("username", username);  // 重构不安全
```

### 分页查询
```java
Page<SysUser> page = new Page<>(current, size);
return userMapper.selectPage(page, wrapper);
```

### 逻辑删除
已全局配置，Entity 字段加 `@TableLogic` 即可，delete 操作自动变为 UPDATE deleted=1。

### 自动填充
`createdAt` 和 `updatedAt` 由 `MyBatisMetaHandler` 自动填充，Entity 字段加注解：
```java
@TableField(fill = FieldFill.INSERT)
private LocalDateTime createdAt;

@TableField(fill = FieldFill.INSERT_UPDATE)
private LocalDateTime updatedAt;
```

## 八、Redis 使用规范

### Key 命名
格式：`{业务}:{对象}:{标识}`

| Key | 示例 | TTL |
|---|---|---|
| `auth:token:{userId}` | `auth:token:1` | 24h |
| `proxy_key:{proxyKey}` | `proxy_key:acg-xxx` | 5min |
| `pricing:{provider}:{model}` | `pricing:openai:gpt-4o` | 30min |
| `budget:tokens:{projectId}:{month}` | `budget:tokens:1:2026-04` | 35d |
| `ratelimit:{key}` | `ratelimit:acg-xxx` | 自动清理 |

### 序列化
全局使用 `GenericJackson2JsonRedisSerializer`，Value 存 JSON，Key 存 String。

## 九、日志规范

### 日志级别
- `ERROR`：系统异常、不可恢复的错误
- `WARN`：可恢复的异常、需要关注的情况
- `INFO`：关键业务操作（登录、Key 生成、代理转发）
- `DEBUG`：调试信息（开发环境开启）

### 使用方式
```java
@Slf4j   // Lombok 注解
public class XxxServiceImpl {
    public void doSomething() {
        log.info("创建项目: projectName={}", request.getProjectName());
        log.error("上游API调用失败: {}", e.getMessage(), e);
    }
}
```

### 敏感信息
- **不要**打印完整的 API Key
- **不要**打印用户密码
- Key 脱敏显示前 8 位 + `****`
