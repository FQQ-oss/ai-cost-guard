### 3. 编译错误修复记录

#### 问题 1：IntelliJ 语言级别默认 Java 5
**现象**：`java: 错误: 不支持发行版本 5`
**原因**：`.idea/misc.xml` 缺少 `languageLevel` 属性，IntelliJ 默认回退到 Java 5
**修复**：
- `misc.xml` 添加 `languageLevel="JDK_17"`
- `compiler.xml` 添加 `PROJECT_BYTECODE_TARGET_LEVEL=17`
- `pom.xml` 显式添加 `maven.compiler.source/target=17`

#### 问题 2：Lombok 包不存在
**现象**：`java: 程序包lombok不存在`
**原因**：Lombok 未下载到本地 Maven 仓库，且 IntelliJ 注解处理未启用
**修复**：
- 创建 `.idea/compiler.xml`，启用 annotation processing
- 使用 Maven 手动下载依赖：`mvn dependency:resolve`
- Maven Wrapper 指向的 3.9.6 版本有问题，改用已下载的 3.9.14

#### 问题 3：缺少 spring-security-crypto
**现象**：`java: 程序包org.springframework.security.crypto.bcrypt不存在`
**原因**：项目只用了 `BCryptPasswordEncoder`，但未引入 Spring Security 依赖
**修复**：pom.xml 添加 `spring-security-crypto`（轻量级，不引入完整 Spring Security）

#### 问题 4：缺少 spring-boot-starter-aop
**现象**：`java: 程序包org.aspectj.lang不存在`
**原因**：审计日志切面 `AuditLogAspect` 依赖 AspectJ，但未引入 AOP starter
**修复**：pom.xml 添加 `spring-boot-starter-aop`

#### 问题 5：Result.ok() 方法不存在
**现象**：`java: 找不到符号 方法 ok()`
**原因**：`Result` 类定义的是 `success()` 方法，`RouteRuleController` 错误使用了 `ok()`
**修复**：`RouteRuleController.java` 中 5 处 `Result.ok()` 全部替换为 `Result.success()`

#### 问题 6：ResultCode.PARAM_ERROR 不存在
**现象**：`java: 找不到符号 变量 PARAM_ERROR`
**原因**：`ResultCode` 枚举中没有定义 `PARAM_ERROR`，其他模块类似场景使用的是 `NOT_FOUND`
**修复**：`RouteRuleServiceImpl.java` 中 2 处 `ResultCode.PARAM_ERROR` 替换为 `ResultCode.NOT_FOUND`
