## 三、代理网关管道设计（核心）

### 责任链模式

```
ProxyController
    └── ProxyPipeline.execute(ProxyContext)
            └── ProxyFilterChain
                  ├── [Order 1]  AuthFilter       — 校验代理 Key，设置项目上下文
                  ├── [Order 2]  RateLimitFilter   — Redis Lua 滑动窗口限流（60 req/min）
                  ├── [Order 3]  QuotaFilter       — 检查月度 Token/金额配额
                  ├── [Order 4]  CacheFilter       — SimHash 语义缓存命中检测
                  ├── [Order 5]  RouterFilter       — SpEL 路由匹配 + Round-Robin 选 Key
                  ├── [Order 6]  ForwardFilter      — WebClient 转发到上游 LLM API
                  └── [Order 100] LogFilter          — 记录日志 + 更新 Redis 计数器
```

### Redis Key 设计

| Key 模式 | 用途 | TTL |
|---|---|---|
| `auth:token:{userId}` | JWT Token（支持强制登出） | 24h |
| `proxy_key:{proxyKey}` | 代理 Key 缓存 | 5min |
| `ratelimit:{proxyKey}` | 滑动窗口限流（Sorted Set） | 自动清理 |
| `budget:tokens:{projectId}:{yyyy-MM}` | 月度 Token 计数器 | 35d |
| `budget:amount:{projectId}:{yyyy-MM}` | 月度金额计数器 | 35d |
| `stats:{projectId}:{yyyy-MM-dd}:{model}` | 每日统计 Hash | 2d |
| `sem_cache:{projectId}:{simHash}` | 语义缓存响应 | 可配置 |
| `sem_index:{projectId}` | SimHash 索引 Set | - |
| `pricing:{provider}:{model}` | 模型定价缓存 | 30min |
| `cache_config:{projectId}` | 缓存配置 | 5min |

### 请求转发流程

```
客户端 → POST /api/v1/chat/completions
         Header: Authorization: Bearer acg-xxxxxxxx
         Body: { model: "gpt-4o", messages: [...] }
              │
              ▼
         AuthFilter: 查 Redis/DB 验证 acg-key → 得到 projectId
              │
              ▼
         RateLimitFilter: Redis ZRANGEBYSCORE 滑动窗口检查
              │
              ▼
         QuotaFilter: Redis GET budget:tokens/amount 检查配额
              │
              ▼
         CacheFilter: 计算 SimHash → 查 Redis sem_index → 命中则跳过转发
              │
              ▼
         RouterFilter: SpEL 匹配规则 → 选 provider/model → Round-Robin 选 Key
              │
              ▼
         ForwardFilter: WebClient POST → 上游 API → 解析 usage → 计算 cost
              │
              ▼
         LogFilter: 写 request_log + Redis INCRBY 计数器
              │
              ▼
         返回 ChatResponse
```

## 五、安全设计

| 层 | 机制 |
|---|---|
| 认证 | JWT（HMAC-SHA256），24h 过期，Redis 存储支持强制登出 |
| 密码 | BCrypt 哈希（spring-security-crypto） |
| API Key 存储 | AES/ECB 加密，密钥 `AiCostGuard@2025` |
| 接口保护 | JwtAuthInterceptor 拦截 /api/**，排除 login/register/proxy |
| CORS | 允许所有源（开发环境），生产需收紧 |
| 前端 | Token 存 localStorage，Axios 拦截器自动附加 Bearer |
| 日志脱敏 | Key 池列表仅显示前 8 位 + **** |

## 六、定时任务

| 任务 | 频率 | 功能 |
|---|---|---|
| StatsFlushTask | 每 5 分钟 | 将 Redis 实时统计数据 upsert 到 MySQL usage_stats_daily 表 |
| AlertCheckTask | 每 1 分钟 | 遍历所有启用的告警规则，检查预算/突增阈值，触发告警通知 |
